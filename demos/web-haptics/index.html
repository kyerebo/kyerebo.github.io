Web Haptics API

Test bed for the proposed Web Haptics API — a semantic, cross-platform interface for tactile feedback. Two mechanisms: the imperative navigator.playHaptics(effect, intensity) and the declarative CSS @haptic at-rule.

Checking for API support…

Imperative API (JavaScript)

navigator.playHaptics()

Each button calls navigator.playHaptics(effect, intensity) with one of the four predefined effects. Requires sticky user activation — a click satisfies that. If the last input device isn't haptics-capable, nothing plays (and, by design, the API returns undefined either way).

1.00
// Activity log
navigator.playHaptics("align", 0.8);

Declarative API (CSS)

The @haptic at-rule

The nested @haptic <effect> <intensity>? at-rule fires once when its containing rule starts matching an element — no JavaScript in the critical path. Interact with the examples below to feel each one.

1. Press confirmation

A tactile align when an "Add to cart" button is pressed, fired from :active.

.add-to-cart:active {
  scale: 0.95;
  @haptic align 0.8;
}

2. Toggle switch

A tick on both on and off. Per-rule tracking means the two transitions fire independently — no collision.

input:checked + .toggle-track       { @haptic tick; }
input:not(:checked) + .toggle-track { @haptic tick; }

3. Focus hint & validation edge

A light hint when the field gains focus, and a heavy edge if validation fails. Type something that isn't an email and press Validate.

.field:focus   { @haptic hint 0.5; }
.field.invalid { @haptic edge; }

4. Scroll-snap carousel

A tick as each slide snaps into place, using the draft :snapped pseudo-class. Swipe or drag horizontally.

5. Keyframe choreography

@haptic nested inside @keyframes enables multi-step haptic choreography — an edge at impact, then a softer align on settle.

@keyframes bounce-settle {
  0%   { transform: translateY(-100%); }
  40%  { transform: translateY(0);   @haptic edge; }
  60%  { transform: translateY(-20%); }
  100% { transform: translateY(0);   @haptic align 0.5; }
}
